# SCADA WATER SYSTEM

Учебный промышленный симулятор, моделирующий систему водоподготовки с управлением в реальном времени.
Проект моделирует работу промышленного водяного узла с насосами, клапанами и резервуарами, с физической моделью напора, потока и уровня воды.

Система построена на data-driven FSM архитектуре с расширяемыми behaviors и сценарной системой через ScriptableObject.

## Цель проекта:
- Создать промышленный тренажёр для моделирования работы операторов SCADA-систем;
- Отработать масштабируемую архитектуру устройств на базе FSM + Behavior;
- Обеспечить лёгкое расширение системы без изменения ядра.

## Архитектура устройств
Система построена на Finite State Machine (FSM) архитектуре:
```
Device
 ├── StateMachine
 │     ├── State (Idle, Running и др.)
 │            └── Behaviors (FlowBehavior …)
```

### Принципы работы:
- State управляет жизненным циклом behaviors (включает/выключает).
- Behavior содержит только прикладную логику (например, расчёт напора или потока).
- Переходы между состояниями изолированы внутри State, без вмешательства других компонентов.

## Устройства

| Устройство    | Описание                                                                     |
| ------------- | ---------------------------------------------------------------------------- |
| **Насос**     | Управляет напором и подачей воды. |
| **Клапан**    | Регулирует поток, может быть открытым или закрытым.                          |
| **Резервуар** | Симулирует накопление и расход воды, уровень обновляется в реальном времени. |

Каждое устройство содержит:

- Изолированные классы состояний с явной логикой переходов;
- Расширяемые behaviors;
- Настраиваемые параметры через инспектор.

> [!NOTE]
> Почему FSM + Behavior?
> - Изоляция логики состояний и прикладной логики
> - Возможность повторного использования behaviors между устройствами
> - Лёгкое масштабирование системы и добавление новых устройств

### Расширяемость

Чтобы добавить новое устройство:
1. Создать enum состояний
2. Реализовать классы состояний
3. Создать behaviors для нового устройства

> Система не требует изменений базового FSM-кода и поддерживает динамическую загрузку новых устройств.

## PropertyDrawer для Behaviors
Для удобного редактирования behaviors в инспекторе Unity используется кастомный PropertyDrawer.
- Позволяет выбрать тип behavior из выпадающего списка.
- Автоматически создаёт экземпляр выбранного типа.
- Поддерживает вложенные поля behavior для редактирования прямо в инспекторе.

### Сценарная система
- Сценарии задаются через ScriptableObject и позволяют:
- Описывать начальное состояние системы
- Задавать цели оператору
- Проверять корректность действий
